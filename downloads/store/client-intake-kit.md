# Client Intake Kit

Copy these prompts into your form, email, or project board before starting a small client project.

## Project Basics

- What are we building?
- What problem should this solve?
- Who is the main audience?
- What should a visitor or user do first?
- What deadline matters, and why?

## Scope

- What pages, screens, commands, or core features are required?
- What would be nice to have but is not required for launch?
- Are there examples of similar work you like?
- Are there examples you specifically want to avoid?
- Who approves the final result?

## Assets

- Do you already have a logo?
- Do you have brand colors, fonts, or a style guide?
- Do you have copy ready?
- Do you have images, icons, files, or product data ready?
- Are there accounts or API keys needed for setup?

## Technical Notes

- Where should this be hosted?
- Does it need a custom domain?
- Does it need analytics?
- Does it need payments, forms, or email notifications?
- Are there privacy, accessibility, or compliance needs?

## Follow-Up Prompts

- What would make this project feel successful one month after launch?
- What should be easy for you to update later?
- What should I explain or document when the project is delivered?
