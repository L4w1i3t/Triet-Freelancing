# Mini Launch Checklist

Use this as a final pass before publishing a small digital project.

## Content

- Confirm the headline names the offer or project clearly.
- Read every visible sentence out loud once.
- Check every button and link destination.
- Confirm contact details and social links are current.
- Add alt text to every meaningful image.

## Usability

- Test the page at desktop, tablet, and mobile widths.
- Confirm form fields have labels and useful placeholders.
- Make sure tap targets are easy to press on mobile.
- Check that important text is not hidden behind fixed headers.
- Confirm keyboard focus is visible.

## Performance

- Compress large images before launch.
- Remove unused scripts and embeds.
- Prefer system behavior over heavy custom animation.
- Test with a slow network profile if possible.
- Make sure the first screen renders without waiting on noncritical scripts.

## Search And Sharing

- Add a clear page title.
- Add a concise meta description.
- Check Open Graph title, description, and image.
- Confirm the favicon loads.
- Submit or refresh the sitemap if the site has one.

## After Launch

- Visit the live URL in a private browser window.
- Send a test contact form submission.
- Save a dated copy of launch notes.
- Schedule a quick check-in one week after launch.
